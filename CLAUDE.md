# Как использовать этот MCP в Claude

## Что делает этот сервер

Он не устраивает спор между моделями.

Он просто:

1. отправляет один и тот же запрос в `Grok`, `Gemini` и `DeepSeek`
2. получает по одному короткому ответу от каждой модели
3. возвращает компактный результат тебе

Ты как Claude уже сам делаешь краткую итоговую выжимку пользователю.

## Когда его вообще вызывать

Вызывай этот сервер, только если задача реально спорная или дорогая по цене ошибки.

Подходит:

- выбор стратегии
- сравнение продуктов или моделей
- red team разбор идеи
- сложный бизнес-вопрос
- продуктовые и контентные решения

Не подходит:

- “что это значит?”
- “перескажи статью”
- “какая сегодня новость?”
- любой простой факт

## Как формулировать запрос

Хорошо:

- чёткая задача
- 1 главный вопрос
- контекст отдельным блоком
- явный критерий успеха
- формат ответа короткий

Плохо:

- длинная простыня без приоритета
- 7 вопросов в одном запросе
- абстрактное “подумайте на тему”

## Два режима

### china

- `moonshotai/kimi-k2.5`
- `qwen/qwen3.6-plus`
- `deepseek/deepseek-v3.2`

### usa

- `google/gemini-3.1-flash-lite-preview`
- `x-ai/grok-4.1-fast`
- `openai/gpt-5.4-nano`

## Роли моделей

### Grok / Kimi

Сильные стороны:

- быстро формирует сильную позицию
- может дать яркий угол зрения
- полезен там, где нужен смелый тезис

Слабые стороны:

- может быть слишком уверенным
- иногда перегибает с категоричностью

Лучше просить:

- занять позицию
- назвать сильный тезис
- предложить ход или гипотезу

### Gemini / Qwen

Сильные стороны:

- структура
- аккуратный синтез
- хороший структурный вариант

Слабые стороны:

- может быть слишком ровным и осторожным
- иногда теряет остроту позиции

Лучше просить:

- сравнить варианты
- дать структурный ответ
- собрать аккуратную рамку решения

### DeepSeek / GPT

Сильные стороны:

- логика
- критика
- вскрытие слабых мест

Слабые стороны:

- может быть слишком сухим
- иногда уходит в “слишком умный” разбор

Лучше просить:

- найти дыры в аргументации
- сделать red team
- показать, где решение ломается

В tools теперь есть параметр `preset`.

Обычно:
- `preset="china"` для китайской тройки
- `preset="usa"` для американской тройки

## Как экономить контекст

Всегда проси:

- “коротко”
- “без вступления”
- “только главное”
- “не повторяй вопрос”
- “дай verdict, 3 аргумента, 3 риска, recommendation”

Если нужна глубина, лучше звать `tri_model_red_team` или `tri_model_compare`, а не раздувать один огромный prompt.

## Базовый шаблон

```text
Задача: [одна задача]

Контекст:
- ...
- ...

Критерий успеха:
- ...

Формат:
- короткий verdict
- 3 key points
- 2 risks
- практический вывод
```
